# Git & GitHub Learning

This repository documents my journey of learning **Git** and **GitHub** through hands-on practice. Instead of only studying commands, I'm performing real version control operations to understand how Git works in day-to-day development.

## What I'm Practicing

* Repository Initialization (`git init`)
* Cloning Repositories (`git clone`)
* Staging Changes (`git add`)
* Creating Commits (`git commit`)
* Branching & Switching (`git branch`, `git checkout`)
* Merging Branches (`git merge`)
* Pushing to Remote (`git push`)
* Pulling Updates (`git pull`)
* Resolving Merge Conflicts
* Working with Remote Repositories
* GitHub Workflow & Best Practices

## Purpose

* Understand Git fundamentals through practical experience.
* Learn professional version control workflows.
* Build confidence using Git and GitHub for real-world projects.
* Maintain a record of commands, experiments, and learning progress.

This repository will continue to grow as I explore more Git concepts, workflows, and best practices.
